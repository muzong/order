## 一、关键业务算法

### 1.1 订单金额计算

> **描述**：根据用户购买的商品清单，计算订单的最终应付总额。

**输入**：
- `order_items`: 订单商品项列表（包含：SKU_ID、单价、购买数量）

**输出**：
- `total_amount`: 订单总金额（DECIMAL(18,2)）

**计算规则**：
1. 遍历所有订单商品项
2. 每项小计 = 单价 × 购买数量
3. 订单总额 = Σ(每项小计)
4. 约束：订单总额不能为负数，最低为 0.00

**伪代码**：
```
FUNCTION calculateOrderAmount(List<OrderItem> items) RETURNS BigDecimal
    totalAmount = 0.00
    
    FOR EACH item IN items:
        itemTotal = item.unitPrice.multiply(BigDecimal.valueOf(item.quantity))
        totalAmount = totalAmount.add(itemTotal)
        
        // 累加过程中的防御性检查
        IF itemTotal.isNegative():
            THROW "商品小计不能为负数"
        END IF
        
        IF item.unitPrice.isNegative():
            THROW "商品单价不能为负数"
        END IF
        
        IF item.quantity <= 0:
            THROW "购买数量必须大于0"
        END IF
    END FOR
    
    // 最终校验
    IF totalAmount.compareTo(BigDecimal.ZERO) < 0:
        totalAmount = BigDecimal.ZERO
        
    RETURN totalAmount.setScale(2, RoundingMode.HALF_UP)
END FUNCTION
```

### 1.2 库存预占逻辑

> **描述**：为订单预占商品库存，防止超卖。

**输入**：
- `sku_id`: 商品 SKU 编号
- `quantity`: 需要预占的数量
- `lock_id`: 预占锁标识（幂等性要求）

**输出**：
- `success`: Boolean（预占是否成功）
- `error_msg`: String（失败原因）

**计算规则**：
1. 从数据库查询当前 SKU 的库存信息
2. 计算可用库存 = 总库存 - 预占库存 - 已售库存
3. 判断可用库存 >= 预占数量
4. 如果充足，则执行原子更新：
   - locked_stock = locked_stock + quantity
5. 更新成功返回 true，否则返回 false

**伪代码**：
```
FUNCTION lockInventory(String sku_id, Integer quantity, String lock_id) RETURNS Boolean
    // 1. 获取分布式锁（防止并发预占）
    lockKey = "inventory:lock:" + sku_id
    IF NOT redisLock.tryLock(lockKey, 30, SECONDS):
        ERROR "并发冲突：库存服务繁忙"
        RETURN false
    END IF
    
    TRY:
        // 2. 查询数据库（开启事务）
        inventory = db.selectFrom("inventory")
            .where("sku_id = ?", sku_id)
            .and("is_deleted = 0")
            .forUpdate()
            
        IF inventory IS NULL:
            ERROR "商品库存记录不存在"
            RETURN false
        END IF
        
        // 3. 计算可用库存
        availableStock = inventory.total_stock 
                       - inventory.locked_stock 
                       - inventory.sold_stock
        
        // 4. 判断库存是否充足
        IF availableStock < quantity:
            ERROR "库存不足：可用库存=" + availableStock + "，需求数量=" + quantity
            RETURN false
        END IF
        
        // 5. 执行原子更新（使用乐观锁）
        rows_affected = db.update("inventory")
            .set("locked_stock", inventory.locked_stock + quantity)
            .where("sku_id = ?", sku_id)
            .and("version = ?", inventory.version)  // 乐观锁版本号
            .execute()
            
        IF rows_affected == 0:
            ERROR "并发冲突：库存记录已被其他事务更新"
            RETURN false
        END IF
        
        // 6. 记录预占日志（用于回滚）
        db.insertInto("inventory_lock_record")
            .set("sku_id", sku_id)
            .set("quantity", quantity)
            .set("lock_id", lock_id)
            .set("created_at", NOW())
            .execute()
            
        RETURN true
        
    CATCH Exception e:
        ERROR "库存预占异常：" + e.getMessage()
        RETURN false
        
    FINALLY:
        redisLock.unlock(lockKey)
    END TRY
END FUNCTION
```

### 1.3 库存释放逻辑

> **描述**：订单取消或支付失败时，释放预占的库存。

**输入**：
- `sku_id`: 商品 SKU 编号
- `quantity`: 需要释放的数量
- `lock_id`: 预占锁标识

**输出**：
- `success`: Boolean（释放是否成功）

**计算规则**：
1. 校验预占记录是否存在（lock_id）
2. 执行原子更新：
   - locked_stock = locked_stock - quantity
   - 如果 locked_stock < 0，则捕获异常
3. 清理预占记录

**伪代码**：
```
FUNCTION releaseInventory(String sku_id, Integer quantity, String lock_id) RETURNS Boolean
    // 1. 验证预占记录
    lockRecord = db.selectFrom("inventory_lock_record")
        .where("sku_id = ?", sku_id)
        .and("lock_id = ?", lock_id)
        .and("is_deleted = 0")
        .one()
        
    IF lockRecord IS NULL:
        ERROR "预占记录不存在"
        RETURN false
    END IF
    
    // 2. 获取分布式锁
    lockKey = "inventory:lock:" + sku_id
    IF NOT redisLock.tryLock(lockKey, 30, SECONDS):
        ERROR "并发冲突：库存服务繁忙"
        RETURN false
    END IF
    
    TRY:
        // 3. 查询当前库存（开启事务）
        inventory = db.selectFrom("inventory")
            .where("sku_id = ?", sku_id)
            .forUpdate()
            
        IF inventory IS NULL:
            ERROR "库存记录不存在"
            RETURN false
        END IF
        
        // 4. 执行原子更新
        newLockedStock = inventory.locked_stock - quantity
        
        IF newLockedStock < 0:
            ERROR "释放失败：预占库存不足（locked_stock=" + inventory.locked_stock + "，需释放=" + quantity + "）"
            ROLLBACK
            RETURN false
        END IF
        
        rows_affected = db.update("inventory")
            .set("locked_stock", newLockedStock)
            .where("sku_id = ?", sku_id)
            .and("version = ?", inventory.version)
            .execute()
            
        IF rows_affected == 0:
            ERROR "并发冲突：库存记录已被更新"
            RETURN false
        END IF
        
        // 5. 标记预占记录已释放
        db.update("inventory_lock_record")
            .set("is_deleted", 1)
            .where("id = ?", lockRecord.id)
            .execute()
            
        RETURN true
        
    CATCH Exception e:
        ERROR "库存释放异常：" + e.getMessage()
        RETURN false
        
    FINALLY:
        redisLock.unlock(lockKey)
    END TRY
END FUNCTION
```

### 1.4 库存扣减确认逻辑

> **描述**：支付成功后，将预占库存正式扣减为已售库存。

**输入**：
- `sku_id`: 商品 SKU 编号
- `quantity`: 需要扣减的数量

**输出**：
- `success`: Boolean（扣减是否成功）

**计算规则**：
1. 原子更新。
2. locked_stock = locked_stock - quantity
3. sold_stock = sold_stock + quantity

**伪代码**：
```
FUNCTION confirmSale(String sku_id, Integer quantity) RETURNS Boolean
    // 1. 获取分布式锁
    lockKey = "inventory:lock:" + sku_id
    IF NOT redisLock.tryLock(lockKey, 30, SECONDS):
        ERROR "并发冲突：库存服务繁忙"
        RETURN false
    END IF
    
    TRY:
        // 2. 查询当前库存（开启事务）
        inventory = db.selectFrom("inventory")
            .where("sku_id = ?", sku_id)
            .forUpdate()
            
        IF inventory IS NULL:
            ERROR "库存记录不存在"
            RETURN false
        END IF
        
        // 3. 校验预占库存是否充足
        IF inventory.locked_stock < quantity:
            ERROR "确认销售失败：预占库存不足（locked_stock=" + inventory.locked_stock + "）"
            RETURN false
        END IF
        
        // 4. 执行原子更新
        rows_affected = db.update("inventory")
            .set("locked_stock", inventory.locked_stock - quantity)
            .set("sold_stock", inventory.sold_stock + quantity)
            .where("sku_id = ?", sku_id)
            .and("version = ?", inventory.version)
            .execute()
            
        IF rows_affected == 0:
            ERROR "并发冲突：库存记录已被更新"
            RETURN false
        END IF
        
        RETURN true
        
    CATCH Exception e:
        ERROR "库存扣减异常：" + e.getMessage()
        RETURN false
        
    FINALLY:
        redisLock.unlock(lockKey)
    END TRY
END FUNCTION
```

## 二、状态机设计

### 2.1 订单状态机

> **描述**：订单在生命周期内的状态流转规则。

**状态定义**：
| 状态码 | 状态名称 | 说明 |
|:------|:---------|:---------|
| 0 | PENDING_PAYMENT | 待付款（初始状态） |
| 1 | PAID | 已支付 |
| 2 | CANCELLED | 已取消 |

**状态流转表**：

| 当前状态 | 触发事件 | 新状态 | 条件 | 备注 |
|:---------|:---------|:-------|:-----|:-----|
| PENDING_PAYMENT | ORDER_CREATE | PENDING_PAYMENT | - | 初始创建订单 |
| PENDING_PAYMENT | PAY_SUCCESS | PAID | 支付回调成功，金额匹配 | 支付成功 |
| PENDING_PAYMENT | PAY_EXPIRED | CANCELLED | 订单超时（15分钟） | 超时自动取消 |
| PENDING_PAYMENT | USER_CANCEL | CANCELLED | 用户主动取消 | 手动取消 |
| PAID | - | PAID | - | 已支付（终态） |
| CANCELLED | - | CANCELLED | - | 已取消（终态） |

**状态图**：
```mermaid
stateDiagram-v2
    [*] --> PENDING_PAYMENT
    
    PENDING_PAYMENT --> PAID: 支付成功\n(订单未超时 + 金额匹配)\n[PaySuccessEvent]
    
    PENDING_PAYMENT --> CANCELLED: 用户取消\n(UserCancelEvent)
    
    PENDING_PAYMENT --> CANCELLED: 订单超时\n(TimedOutEvent)
    
    state PAID {
        [*] --> PAID
    }
    
    state CANCELLED {
        [*] --> CANCELLED
    }
    
    PAID --> [*]
    CANCELLED --> [*]
```

**非法流转拦截**：
- `PAID` → `PENDING_PAYMENT`：禁止（已支付订单不可回退到待付款）
- `CANCELLED` → `PENDING_PAYMENT`：禁止（已取消订单不可恢复）
- `CANCELLED` → `PAID`：禁止（已取消订单不可支付）
- `PAID` → `CANCELLED`：禁止（已支付订单不可取消）

## 三、核心流程伪代码

### 3.1 用户下单主流程

> **描述**：用户提交订单的完整业务逻辑，包含库存校验、预占和订单生成。

**前置条件**：
- 用户已登录（user_id有效）
- 商品 SKU_ID 存在且可售
- 购买数量 > 0

**后置结果**：
- 成功：订单创建成功，库存已预占，返回订单信息
- 失败：库存不足 → 提示"库存不足"；系统异常 → 提示"系统繁忙"

**详细步骤**：
```
FUNCTION createOrder(CreateOrderRequest request) RETURNS CreateOrderResponse
    // 1. 参数校验
    IF request.user_id IS NULL:
        RETURN error("用户ID不能为空")
    END IF
    
    IF request.sku_id IS NULL OR request.sku_id.isEmpty():
        RETURN error("SKU_ID不能为空")
    END IF
    
    IF request.quantity IS NULL OR request.quantity <= 0:
        RETURN error("购买数量必须大于0")
    END IF
    
    // 2. 幂等性校验（可选：根据业务需求）
    orderNo = generateOrderNo()
    IF orderRepository.findByOrderNo(orderNo) IS NOT NULL:
        orderNo = generateOrderNo() // 重新生成
    END IF
    
    // 3. 预占库存（开启独立事务）
    stockLockId = "LOCK:" + orderNo
    IF NOT inventoryService.lock(request.sku_id, request.quantity, stockLockId):
        RETURN error("库存不足，请调整购买数量", code=1001)
    END IF
    
    TRY:
        // 4. 查询商品信息（用于计算金额）
        product = productRepository.findBySkuId(request.sku_id)
        IF product IS NULL:
            THROW "商品不存在"
        END IF
        
        // 5. 计算订单金额
        totalAmount = product.unitPrice.multiply(BigDecimal.valueOf(request.quantity))
        
        // 6. 计算支付截止时间（当前时间 + 15分钟）
        payTimeoutAt = NOW().plusMinutes(15)
        
        // 7. 构建订单对象
        order = Order()
            .setOrderNo(orderNo)
            .setUserId(request.user_id)
            .setTotalAmount(totalAmount)
            .setStatus(0) // PENDING_PAYMENT
            .setPayTimeoutAt(payTimeoutAt)
            .setCreatedAt(NOW())
            
        orderItem = OrderItem()
            .setOrderId(order.id)
            .setSkuId(request.sku_id)
            .setProductName(product.name)
            .setUnitPrice(product.unitPrice)
            .setQuantity(request.quantity)
            .setItemTotal(totalAmount)
            
        // 8. 创建订单（开启本地事务）
        db.beginTransaction()
        TRY:
            orderRepository.save(order)
            orderItemRepository.save(orderItem)
            db.commit()
            
        CATCH Exception e:
            db.rollback()
            ERROR "订单创建失败：" + e.getMessage()
            
            // 9. 回滚库存预占
            inventoryService.release(request.sku_id, request.quantity, stockLockId)
            
            RETURN error("系统繁忙，请稍后重试", code=1003)
        END TRY
        
        // 10. 发布订单创建事件（异步）
        eventPublisher.publish(OrderCreatedEvent(order))
        
        // 11. 返回成功
        RETURN success({
            order_no: orderNo,
            total_amount: totalAmount,
            pay_timeout_at: payTimeoutAt
        })
        
    CATCH Exception e:
        ERROR "下单流程异常：" + e.getMessage()
        RETURN error("系统繁忙，请稍后重试", code=1003)
    END TRY
END FUNCTION
```

### 3.2 支付回调处理流程

> **描述**：支付渠道异步通知支付结果，系统处理回调并更新订单状态。

**前置条件**：
- 支付服务已接收到回调请求
- 回调参数包含：订单号、支付流水号、金额、状态

**后置结果**：
- 成功：订单状态更新为"已支付"，库存扣减确认
- 异常：金额不符 → 标记为异常；订单已取消 → 拒绝处理

**详细步骤**：
```
FUNCTION handlePaymentCallback(PaymentCallback callback) RETURNS CallbackResponse
    // 1. 幂等性校验（Redis Key: payment_callback:{payment_no}）
   幂等Key = "payment_callback:" + callback.payment_no
    IF redis.exists(幂等Key):
        LOG "重复回调（幂等跳过）"
        RETURN success("重复回调已处理")
    END IF
    
    // 2. 设置幂等锁（防止并发回调）
    IF NOT redis.setnx(幂等Key, "1", 24, HOURS):
        RETURN success("并发回调已处理")
    END IF
    
    TRY:
        // 3. 查询支付凭证
        payment = paymentRepository.findByPaymentNo(callback.payment_no)
        IF payment IS NULL:
            ERROR "支付凭证不存在"
            RETURN error("支付流水号不存在", code=3001)
        END IF
        
        // 4. 校验订单状态
        order = orderRepository.findById(payment.order_id)
        
        IF order IS NULL:
            ERROR "订单不存在"
            RETURN error("订单不存在", code=3001)
        END IF
        
        IF order.status == 2: // CANCELLED
            ERROR "订单已取消"
            RETURN error("订单已关闭", code=3004)
        END IF
        
        IF order.status == 1: // PAID
            LOG "订单已支付（幂等跳过）"
            RETURN success("订单已支付")
        END IF
        
        // 5. 校验金额（防止篡改）
        IF callback.amount.compareTo(order.total_amount) != 0:
            ERROR "金额不匹配"
            
            // 修改支付状态为"异常"
            payment.setStatus(3) // EXCEPTION
            paymentRepository.save(payment)
            
            // 发起人工审核通知
            notifyAuditTeam(order, "支付金额不符")
            
            RETURN error("支付金额与订单金额不符", code=3003)
        END IF
        
        // 6. 处理支付成功
        IF callback.status == 1: // SUCCESS
            db.beginTransaction()
            TRY:
                // 更新支付状态
                payment.setStatus(1) // SUCCESS
                payment.setUpdatedAt(NOW())
                paymentRepository.save(payment)
                
                // 更新订单状态
                order.setStatus(1) // PAID
                order.setPaySuccessAt(NOW())
                orderRepository.save(order)
                
                // 库存扣减确认
                inventoryService.confirmSale(order.sku_id, order.quantity)
                
                db.commit()
                
                // 发布支付成功事件（异步）
                eventPublisher.publish(PaymentSuccessEvent(order))
                
                RETURN success("支付成功")
                
            CATCH Exception e:
                db.rollback()
                ERROR "支付成功处理异常：" + e.getMessage()
                RETURN error("系统繁忙，请稍后重试", code=3005)
            END TRY
            
        ELSE IF callback.status == 2: // FAILED
            // 处理支付失败（释放库存）
            payment.setStatus(2) // FAILED
            paymentRepository.save(payment)
            
            inventoryService.release(order.sku_id, order.quantity, stockLockId)
            
            RETURN success("支付失败，库存已释放")
        END IF
        
    CATCH Exception e:
        ERROR "支付回调处理异常：" + e.getMessage()
        RETURN error("系统繁忙，请稍后重试", code=3005)
        
    FINALLY:
        // 移除幂等锁
        redis.delete(幂等Key)
    END TRY
END FUNCTION
```

### 3.3 订单超时取消流程

> **描述**：定时任务扫描超时订单并自动取消。

**前置条件**：
- 定时任务每分钟执行
- 数据库存在超时订单（当前时间 > pay_timeout_at 且 status = 0）

**后置结果**：
- 订单状态更新为"已取消"
- 库存回滚（locked_stock 减少，available_stock 增加）

**详细步骤**：
```
FUNCTION cancelTimeoutOrders()
    // 1. 扫描超时订单（分批处理，每次100条）
    timeoutOrders = db.selectFrom("orders")
        .where("status = 0") // PENDING_PAYMENT
        .and("pay_timeout_at < NOW()")
        .and("is_deleted = 0")
        .limit(100)
        .fetch()
        
    IF timeoutOrders.isEmpty():
        LOG "无超时订单"
        RETURN
    END IF
    
    FOR EACH order IN timeoutOrders:
        TRY:
            db.beginTransaction()
            TRY:
                // 2. 更新订单状态为取消
                db.update("orders")
                    .set("status", 2) // CANCELLED
                    .set("updated_at", NOW())
                    .where("id = ?", order.id)
                    .and("status = 0") // 确保状态未被并发修改
                    .execute()
                
                // 3. 查询订单明细（获取 SKU 和数量）
                orderItems = db.selectFrom("order_items")
                    .where("order_id = ?", order.id)
                    .fetch()
                    
                // 4. 释放每项商品的库存
                FOR EACH item IN orderItems:
                    inventoryService.release(
                        sku_id = item.sku_id,
                        quantity = item.quantity,
                        lock_id = "TIMEOUT:" + order.order_no
                    )
                END FOR
                
                db.commit()
                LOG "订单超时取消成功: order_no=" + order.order_no
                
            CATCH Exception e:
                db.rollback()
                ERROR "订单超时取消失败: order_id=" + order.id + ", error=" + e.getMessage()
            END TRY
            
        CATCH Exception e:
            ERROR "批量处理异常：" + e.getMessage()
        END TRY
    END FOR
    
    // 5. 记录日志
    LOG "本次扫描共处理 " + timeoutOrders.size() + " 个超时订单"
END FUNCTION
```

## 四、边界场景处理

### 4.1 库存不足场景
- **触发条件**：可售库存 < 购买数量
- **处理动作**：
  1. 拒绝订单创建
  2. 清理预占锁（如有）
  3. 返回错误码 1001 + 提示信息
  4. 记录日志

### 4.2 库存并发冲突场景
- **触发条件**：两个请求同时预占同一 SKU 的库存
- **处理动作**：
  1. Redis 分布式锁保证串行化
  2. 数据库乐观锁版本号冲突
  3. 后续请求验证库存时发现不足
  4. 返回"库存不足"（而非"并发冲突"）

### 4.3 金额不符场景
- **触发条件**：支付回调金额 ≠ 订单金额
- **处理动作**：
  1. 拒绝更新订单状态
  2. 将支付凭证状态标记为"异常"
  3. 发起人工审核通知
  4. 记录异常日志

### 4.4 支付回调晚于超时场景
- **触发条件**：订单已取消（库存已释放），收到支付回调
- **处理动作**：
  1. 校验订单状态为"已取消"（status=2）
  2. 拒绝处理回调
  3. 记录异常日志（"订单已关闭"）
  4. 不再修改订单和库存数据

### 4.5 订单状态并发修改场景
- **触发条件**：用户取消订单与支付回调同时触发
- **处理动作**：
  1. 数据库乐观锁（version 字段）
  2. UPDATE ... WHERE version = ? AND status = 0
  3. 影响行数为 0 表示并发冲突
  4. 重试或拒绝操作
