# 接口定义 (API Specification) V1.0

## 接口概览

| 模块 | 基础路径 | 协议/域名 |
| :--- | :--- | :--- |
| 订单模块 | `/api/v1/order` | HTTP/HTTPS (localhost:8080) |
| 支付模块 | `/api/v1/payment` | HTTP/HTTPS (localhost:8080) |

**统一响应格式**：
```json
{
  "code": 0,
  "msg": "success",
  "data": {}
}
```

## 订单模块接口

### 创建订单

> **简要描述**：用户提交订单，系统校验库存并生成订单记录。

* **URL**: `/api/v1/order/create`
* **Method**: `POST`
* **Tags**: [订单模块]

#### 请求参数 (JSON Body)

| 参数名 | 类型 | 必填 | 说明 | 示例 |
|:---------|:-------|:---|:---------|:---------|
| `user_id` | Long | 是 | 用户 ID | 888 |
| `sku_id` | String | 是 | SKU 编号 | "SKU001" |
| `quantity` | Int | 是 | 购买数量 | 2 |

#### 响应参数 (Response)

**状态码 200 - 成功**
```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "order_no": "ORD20260415001",
    "total_amount": 11998.00,
    "pay_timeout_at": "2026-04-15 10:15:00"
  }
}
```

**状态码 200 - 库存不足**
```json
{
  "code": 1001,
  "msg": "商品库存不足，请调整购买数量",
  "data": null
}
```

#### 错误码定义

| 错误码 |	http 状态码 | 说明 | 处理建议 |
|:------|:-----------|:---------|:---------|
| 1001 | 200 | 库存不足 | 提示用户减少购买数量 |
| 1002 | 400 | 请求参数缺失或格式错误 | 校验用户输入 |
| 1003 | 500 | 系统内部错误 | 记录日志并重试 |

---

### 查询订单

> **简要描述**：根据订单号查询订单详细信息。

* **URL**: `/api/v1/order/query`
* **Method**: `GET`
* **Tags**: [订单模块]

#### 请求参数 (Query)

| 参数名 | 类型 | 必填 | 说明 | 示例 |
|:---------|:-------|:---|:---------|:---------|
| `order_no` | String | 是 | 订单编号 | "ORD20260415001" |

#### 响应参数 (Response)

**状态码 200**
```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "order_no": "ORD20260415001",
    "user_id": 888,
    "total_amount": 11998.00,
    "status": 0,
    "pay_timeout_at": "2026-04-15 10:15:00",
    "create_at": "2026-04-15 10:00:00",
    "items": [
      {
        "sku_id": "SKU001",
        "product_name": "iPhone 15",
        "unit_price": 5999.00,
        "quantity": 2
      }
    ]
  }
}
```

#### 状态枚举说明

| Status Code | 枚举名称 | 说明 |
|:------------|:---------|:---------|
| 0 | PENDING_PAYMENT | 待付款 |
| 1 | PAID | 已支付 |
| 2 | CANCELLED | 已取消 |

#### 错误码定义

| 错误码 |	http 状态码 | 说明 | 处理建议 |
|:------|:-----------|:---------|:---------|
| 2001 | 404 | 订单不存在 | 返回空数据 |

---

### 取消订单

> **简要描述**：用户主动取消未支付订单，库存回滚。

* **URL**: `/api/v1/order/cancel`
* **Method**: `POST`
* **Tags**: [订单模块]

#### 请求参数 (JSON Body)

| 参数名 | 类型 | 必填 | 说明 | 示例 |
|:---------|:-------|:---|:---------|:---------|
| `order_no` | String | 是 | 订单编号 | "ORD20260415001" |
| `user_id` | Long | 是 | 用户 ID | 888 |

#### 响应参数 (Response)

**状态码 200 - 取消成功**
```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "order_no": "ORD20260415001",
    "status": 2
  }
}
```

**状态码 200 - 订单已取消**
```json
{
  "code": 2002,
  "msg": "订单已取消或不存在",
  "data": null
}
```

#### 错误码定义

| 错误码 |	http 状态码 | 说明 | 处理建议 |
|:------|:-----------|:---------|:---------|
| 2002 | 200 | 订单已取消或不存在 | 提示用户订单状态 |
| 2003 | 200 | 订单已支付，不可取消 | 提示用户订单已完成 |

## 支付模块接口

### 发起支付

> **简要描述**：用户发起支付请求，生成支付凭证。

* **URL**: `/api/v1/payment/pay`
* **Method**: `POST`
* **Tags**: [支付模块]

#### 请求参数 (JSON Body)

| 参数名 | 类型 | 必填 | 说明 | 示例 |
|:---------|:-------|:---|:---------|:---------|
| `order_no` | String | 是 | 订单编号 | "ORD20260415001" |
| `user_id` | Long | 是 | 用户 ID | 888 |

#### 响应参数 (Response)

**状态码 200**
```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "payment_no": "PAY20260415001",
    "order_no": "ORD20260415001",
    "amount": 11998.00,
    "status": 0
  }
}
```

#### 错误码定义

| 错误码 |	http 状态码 | 说明 | 处理建议 |
|:------|:-----------|:---------|:---------|
| 3001 | 400 | 订单不存在或订单号为空 | 校验订单号有效性 |
| 3002 | 200 | 订单已支付或已取消 | 提示用户订单已完成 |

---

### 支付回调（模拟）

> **简要描述**：模拟支付渠道异步通知支付结果（Mock场景）。

* **URL**: `/api/v1/payment/callback`
* **Method**: `POST`
* **Tags**: [支付模块]

#### 请求参数 (JSON Body)

| 参数名 | 类型 | 必填 | 说明 | 示例 |
|:---------|:-------|:---|:---------|:---------|
| `payment_no` | String | 是 | 支付流水号 | "PAY20260415001" |
| `order_no` | String | 是 | 订单编号 | "ORD20260415001" |
| `amount` | Decimal | 是 | 支付金额 | 11998.00 |
| `status` | Int | 是 | 支付状态：1-成功, 2-失败 | 1 |

#### 响应参数 (Response)

**状态码 200 - 处理成功**
```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "message": "支付结果处理成功"
  }
}
```

**状态码 200 - 金额不符**
```json
{
  "code": 3003,
  "msg": "支付金额与订单金额不符",
  "data": null
}
```

**状态码 200 - 订单已关闭**
```json
{
  "code": 3004,
  "msg": "订单已关闭，支付回调无效",
  "data": null
}
```

#### 错误码定义

| 错误码 |	http 状态码 | 说明 | 处理建议 |
|:------|:-----------|:---------|:---------|
| 3003 | 200 | 支付金额与订单金额不符 | 触发人工审核流程 |
| 3004 | 200 | 订单已关闭，支付回调无效 | 记录异常日志 |
| 3005 | 500 | 支付回调幂等性校验失败 | 记录重放攻击日志 |
